### ROLE
Atue como Engenheiro Frontend Mobile Sênior, especialista em React Native e arquiteturas escaláveis. Sua missão é implementar o fluxo de Carrinho de Compras seguindo padrões de Clean Code, acessibilidade (A11y) e a estrutura de pastas estrita do projeto.

### TECH STACK & PATTERNS
- Framework: React Native (Expo Router)
- Styling: NativeWind (Tailwind CSS)
- State Management: Zustand (com persistência)
- Icons: Lucide React
- Forms: React Hook Form + Zod
- API Layer: Orval generated clients (localizados em @/src/api/generated)

### SCOPE: CARRINHO DE COMPRAS
Você deve entregar 3 partes fundamentais integradas:

1. STORE (Zustand):
   - Criar `useCartStore` em `@/src/store/use-cart-store.ts`.
   - Implementar persistência usando `createJSONStorage` com `AsyncStorage`.
   - Ações: `addItem`, `removeItem`, `updateQuantity`, `applyCoupon`, `clearCart`.
   - Lógica de Identificação: O badge deve contar o total de itens (somando quantidades e variações).

2. COMPONENTS (UI/UX):
   - CartBadge: Refatorar o componente `<Pressable>` fornecido para exibir o badge numérico sobre o ícone `ShoppingCart`.
   - CartItem: Card de produto individual baseado no Figma.
   - CouponSection: Input de cupom usando React Hook Form, disparando o client do Orval para validar o código.

3. SCREEN (@/app/(public)/(tabs)/cart.tsx):
   - Implementar a listagem, cálculo de subtotal, frete (se houver) e total.
   - Navegação: O ícone da Home deve usar `router.push("/cart")`.

### BUSINESS RULES & CONSTRAINTS
- [REGRA 01]: IDs Únicos. Um produto com variações diferentes (Ex: Camisa P e Camisa M) deve ser tratado como itens distintos no array do carrinho.
- [REGRA 02]: Persistência. O carrinho não pode ser perdido ao fechar o app.
- [REGRA 03]: Integração API. Use EXCLUSIVAMENTE os hooks gerados pelo Orval para validar cupons e buscar detalhes de preço. Não faça fetch manual.
- [REGRA 04]: Feedback Visual. Exiba estados de 'loading' ao validar cupom e 'error message' amigável caso o cupom seja inválido.

### OUTPUT EXPECTED
1. Código do `use-cart-store.ts`.
2. Código do componente `CartBadge` (acessível e responsivo).
3. Código da Screen `cart.tsx` integrando os componentes e a lógica de cupom.
4. Estrutura de pastas onde cada arquivo deve ser alocado.
