# Nero Mobile — Code Style & Conventions

## Component Patterns
- Components use **named exports** (e.g., `export function ProductCard(...)`)
- Screen components use **default exports** (e.g., `export default function HomeScreen()`)
- One component per file as a rule; modular components go in `@/src/components/ui/`
- Gluestack UI primitives are wrapped in `src/components/gluestack/ui/` 
- Use Gluestack wrappers: `Box`, `Text`, `HStack`, `VStack`, `Pressable`, `Image`, `SafeAreaView`

## Styling
- **NativeWind** (TailwindCSS for React Native) with `className` props
- Use Tailwind tokens from `tailwind.config.js` — avoid arbitrary values
- `cn()` utility from `@gluestack-ui/utils/nativewind-utils` for conditional classes
- Inline styles only when necessary (e.g., `contentContainerStyle` on FlatLists)

## Imports
- Path alias: `@/src/...` maps to `src/`
- Imports are organized: external first, then internal by path alphabetically

## API Layer
- **Never edit** files in `src/api/generated/` — they are auto-generated by Orval
- Custom hooks wrap generated hooks (e.g., `useInfiniteProducts` wraps `searchProducts`)
- API types come from `@/src/api/generated/model`

## State Management
- Zustand stores in `src/store/` with `use-*.store.ts` naming
- TanStack Query for server state

## Naming Conventions
- Files: kebab-case (e.g., `search-bar.tsx`, `use-infinite-products.ts`)
- Components: PascalCase (e.g., `SearchBar`, `ProductCard`)
- Hooks: camelCase with `use` prefix (e.g., `useInfiniteProducts`)
- Stores: `use-*-store.ts` (e.g., `use-search-store.ts`)

## Language
- UI text is in **Portuguese (Brazilian)** 
- Code (variables, comments, etc.) is in English
